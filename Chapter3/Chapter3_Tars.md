> 역할, 협력, 책임

객체지향 설계의 핵심은 협력을 구성하기 위해 적절한 객체를 찾고 적절한 책임을 할당하는 것이다. 

### 협력

> 협력은 객체를 설계하는데 필요한 문맥이다.

객체들은 자율적이다. 그리고 다른 객체와 협력하는 사회적인 존재다.

다만 다른 객체의 상세한 구현 내부에는 접근하지 못하기 때문에 협력을 수행하기 위해 메시지를 보내서 자신의 요청을 전송할 수 있다.
외부의 객체는 오직 메시지를 전송할 수 있을 뿐, 메시지를 어떻게 처리할지는 메시지를 수신한 객체가 결정한다. 

객체의 자율성을 보장하기 위해서는 필요한 정보와 정보에 기반한 행동을 객체 안에 모아놔야 한다.
객체를 자율적으로 만드는 가장 기본적인 방법은 내부 구현을 캡슐화 하는 것이다.

협력이 존재하기 때문에 객체가 존재한다.
객체의 행동을 결정하는 것이 협력이고, 객체의 상태를 결정하는 것은 행동이다.
객체의 상태는 행동을 수행하는데 필요한 정보가 무엇인지로 결정된다.

### 책임
> 협력에 참여하기 위해 객체가 수행하는 행동

객체에 의해 정의되는 행위의 집합으로
- 객체가 유지해야 하는 정보(아는 것: knowing)
- 수행할 수 있는 행동(하는 것: doing)

협력 안에서 객체에게 할당한 책임이 외부의 인터페이스와 내부의 속성을 결정한다.
중요한 건, 아는 것과 하는 것이 밀접하게 연관되어 있다는 점이다.

객체는 자신이 맡은 책임을 수행하는데 필요한 정보를 알 책임이 있다.
객체는 자신이 할 수 없는 일을 도와줄 객체를 알고 있을 책임이 있다.

협력이 중요한 이유는 객체에게 할당할 책임을 결정할 문맥을 제공하기 때문이다.

객체에게 책임을 할당하기 위해서 협력이라는 문맥을 정의한다.

객체가 책임을 수행하게 하기 위해서는 메시지를 전송해야 한다.
책임 할당은 전송할 메시지의 이름을 정하는 것과 같다.
### 책임 할당

#### 정보 전문가 패턴
협력에 필요한 지식과 방법을 가장 잘 알고 있는 객체에게 도움을 요청한다.

책임을 수행하는데 필요한 정보를 충분히 알고 있지 않다면 가장 잘 아는 객체에게 도움을 요청해보자

#### 책임 주도 설계
책임을 찾고 책임을 수행할 적절한 객체를 찾아 책임을 할당하는 방식

RDD(Responsibility-Driven-Design) 

메시지가 객체를 선택하게 해야한다.

#### 행동이 상태를 결정한다.
객체가 협력에 적합한 지 결정하는 것은 객체의 상태가 아닌 행동이다.
객체의 상태(내부구현)에 초점을 맞추지 말라. 캡슐화를 저해한다.

캡슐화를 지키기 위해서는 항상 협력이라는 문맥 안에서 객체를 생각해야 한다.
상태는 단지 객체가 행동을 정상적으로 수행하기 위한 재료일 뿐이다.

### 역할
> 객체가 특정 협력 안에서 수행하는 책임의 집합 

역할에게 책임을 할당한다.
적절한 역할을 찾고, 역할을 수행할 객체로 인스턴스를 선택한다.
여러 종류의 객체들이 협력에 참여할 수 있으면 역할이라고 부르면 된다.
